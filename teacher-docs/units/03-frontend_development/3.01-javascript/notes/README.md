# JS Intro

## General JS

- JavaScript is NOT the same Java
- script tag - links JS to an HTML file
- alert - pop up for users
- console.log - used by devs for debugging
- comments - lines of code that won't run
- pseudocode - writing a step by step process in plain english
- variables - placeholder for values
  - declaring a variable (be a good dev)
    - var - DO NOT USE (old syntax)
    - const - variables that don't change (ALWAYS start off your variables as a const)
    - let - variables that change
- data types
  - number - strictly numbers
  - string - any characters within double quotes, single quotes or back-ticks
  - boolean - true or false
  - null - absolutely nothing
  - undefined - variable that has a name, but not a value yet
  - object - data storage in a key/value pair

# Data Type Details

- Numbers
  - mathematical operations
    - \+ -> add
    - \- -> subtract
    - \* -> multiplication
    - / -> divide
    - % -> modulus (remainder if you divide two numbers)

- Strings
  - concatenate - bringing two things together
    - use the + symbol between two strings (old way)
    - template literals -> back-ticks (variable => ${variableName}, no need for +)
- typeof -> returns the data type as a word (string)

# EXTRA FOR TOMORRROW'S CONDITIONALS DEMO

## Logical operators - always come out as booleans (true or false)

- && -> and (means both are true)
- || -> or (at least one side is true)
- ! -> not (opposite of whatever it is describing)

## Comparison Operators - always comes out as a boolean (true or false)

- == -> loosely equals (DO NOT USE) (checks value only)
- === -> strictly equals (checks data type AND value)
- != -> loosely does not equal (DO NOT USE) (checks value only)
- !== -> strictly does not equal (checks data type AND value)
- \> -> greater than
- < -> less than
- \>= -> greater than or equal to
- <= -> less than or equal to
