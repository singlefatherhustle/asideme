# Semantic HTML

## Syntax

**Syntax** refers to the *rules* and *structure* of how something is written.

- Use angle brackets to denote a tag `<>`
- Use a slash to denote a closing tag `</>`
- Check for typos

## Semantics

**Semantics** refers to the *meaning* of what is written.

In the context of HTML, this means *choosing the correct tag*.

- Use `<p>` for paragraphs
- Use `<section>` to group related content
- Use `<em>` to emphasize words within a paragraph
- Use `<address>` for contact information
- And so on!

## Summary

We need both in order to write good HTML!
