# Data structures

An ADT provides a blueprint, but it still needs to be built.
A **data structure** is an *implementation* of an abstract data type.

Just like how pseudocode can be translated into different code or different languages,
an abstract data type can be implemented by different data structures.

In conversation, we usually just say "data structures" when referring to
either of these ideas.
