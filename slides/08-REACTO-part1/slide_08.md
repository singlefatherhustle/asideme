# What do we use instead?

When analyzing an algorithm, we think about the **worst-case time complexity**, which is the *maximum* number of operations required for an input of a given size.

We use **big O notation** to denote the *asymptotic upper bound* of an algorithm's runtime.

An algorithm's time complexity T_a(n) is big O g(n)

```
T_a(n) = O(g(n))
```

if there exists a positive real number M and a real number n_0 such that

```
|T_a(n)| ≤ M|g(n)| for all n ≥ n_0
```
