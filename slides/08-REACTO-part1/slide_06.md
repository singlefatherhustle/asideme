# Which algorithm is faster?

## Time Complexity Analysis

**T_x(n) =** operations performed by algorithm x for some input of size n

**T_a(n) = 3n - 2**

**T_b(n) = 2n + 5**

## Algorithm Comparison

Algorithm b is **faster** because the number of operations increases **slower** as the input size increases.

Algorithm a is **slower** because the number of operations increases **faster** as the input size increases.

## Time Complexity Graph

The graph displays a comparison of two algorithms with:
- **X-axis:** Input size (0 to 100)
- **Y-axis:** Number of operations (0 to 300)
- **Line a(n):** Orange line representing T_a(n) = 3n - 2
- **Line b(n):** Blue line representing T_b(n) = 2n + 5

At input size 100:
- Algorithm a performs approximately 300 operations
- Algorithm b performs approximately 200 operations
