# How does this help us?

Using big O notation to classify an algorithm's time complexity allows us to **approximate** the number of operations without actually counting them.

If an algorithm has a polynomial time complexity *pn³ + qn² + rn + s*, we can drop all lower order terms and coefficients and just say that the algorithm's run time is *O(n³)*, or "order n cubed".
