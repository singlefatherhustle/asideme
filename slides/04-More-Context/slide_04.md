# Review

## What is context?

Context lets a parent component (a provider) provide data to the entire tree below it.

## Diagram: Context Data Flow

```
Provider (Parent)
    |
    ├─ Connected child component
    ├─ Connected child component
    ├─ Connected child component
    |
    ├─── Nested child component
    └─── Nested child component
```

The diagram shows a parent component at the top with a provider icon distributing data (shown with light orange/tan connecting lines) to three direct child components, and further down to two nested child components.
