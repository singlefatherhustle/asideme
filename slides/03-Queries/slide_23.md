# Always catch your errors

How do you stop a thrown error from crashing a program? By **catching** the error before it's too late (like *catching a thrown* baseball before it hits a window).

## Diagram: Error Catching Flow

```
Developer (angry emoji) 
  ↓ throw
Error (pink circle)
  ↓ (hand stop gesture)
  catch
Friend (happy emoji) → "I have cold brew in the fridge :)"
  ↓ (dashed line)
Window/Door (wooden window illustration)
```

**Flow Description:**
- Developer throws an Error
- Friend catches the Error (preventing it from reaching the window)
- Friend responds positively: "I have cold brew in the fridge :)"
- Error is handled before it crashes the program
