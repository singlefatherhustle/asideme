# Proper syntax isn't enough

Communicating via HTTP by sending a properly formatted request is *just the first step*.

Just like code, the request format indicates the *syntax* of a message, but the message must also have appropriate *semantics*.

In other words, a client can send as many requests as it wants, but it will only get a successful response if the server accepts the request.
