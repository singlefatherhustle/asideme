# How does a client know what to send?

A server that is accepting requests will expose an **API**, which stands for **Application Programming Interface**, which will (hopefully) be documented.

An API opens an *application* to interactions from the outside by allowing communication with *programs* across a mutually agreed-upon *interface*.

## Diagram: Server-Client API Communication

```

│     server      │         │     client      │

```
