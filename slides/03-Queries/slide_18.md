# Can we do something else while we wait?

If a task might take a really long time, then wouldn't it be more efficient to multitask and work on something else? This is a core motivation behind the broader programming concept of **synchronous vs asynchronous** programming.

A **synchronous** program executes one line at a time, in the order it's written, and waits for each line to finish before proceeding to the next line.

An **asynchronous** program can start a long-running operation and continue to the next line *before* the operation finishes.
