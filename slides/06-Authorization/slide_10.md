# Send a token

Once we've **authenticated** the user's identity, we want to hand them a **token** so that they don't need to keep sending their credentials over and over again.

## Diagram: Token Flow

Three components connected by arrows:

- **API** (right) — illustrated as a blue sphere with two protruding connectors
- **token** (center) — illustrated as a gray rectangular key
- **client** (left) — illustrated as a desktop computer with monitor and keyboard

Arrow direction: **API → token → client**

(The token flows from the API to the client)
