# useEffect to the rescue

Yup, React components shouldn't be making API calls! Instead, a component can hook into a React feature that **is** allowed to have side effects.

**useEffect** is a hook that serves as an "escape hatch" from React. It specializes in dealing with side effects that involve *external systems*, such as communication with an API.
