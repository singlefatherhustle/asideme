# It is difficult to write comprehensive tests.

As you may have noticed, even a simple function like `multiply` might end up having many, many test cases. It's very easy for an edge case to slip by!

Even if all of the test cases pass, it is not guaranteed that the code is bug-free.

TDD is not about writing **all** of the test cases at once or getting the code perfect right away. It is about writing **small** tests to make **consistent** and **incremental** progress.
