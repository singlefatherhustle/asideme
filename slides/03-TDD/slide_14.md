# What is test-driven development?

**Test-driven development (TDD)** is a software development process that involves writing tests for your code *before* writing the code itself.

## TDD Steps

1. Write a test for a new feature.
2. Run all tests. The new test should *fail* for *expected* reasons.
3. Write the *simplest* code that passes all tests.
4. Refactor as needed while ensuring all tests continue to pass.
5. Repeat.

## TDD Cycle

The TDD Cycle diagram shows three connected stages in circular flow:

1. **Write a failing test** (red circle) — Starting point where a new test is created that fails
2. **Make the test pass** (green circle) — Write code to make the failing test pass
3. **Refactor** (purple circle) — Improve and refactor the code while maintaining test success

The cycle flows: Write a failing test → Make the test pass → Refactor → (repeat)
