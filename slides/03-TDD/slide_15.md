# What is a test?

A **test specification** ("spec") is an *explicit* expectation of how the code should behave.
A **test suite** is a *group* of expectations about what the code should do.

In other words, TDD requires you to know *what* you want the code to do *before* you write the code. This is one of the *most important skills* that a developer should have!
