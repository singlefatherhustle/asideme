# Commit Messages

Each commit has a **hash** and a **message**. The **hash** is a unique identifier for that commit. A good commit **message** clearly and specifically describes how the repository has changed since the last commit.

```bash
git commit -m "message"
```

## Good Commit Messages

- Fix phone number typo in footer
- Update login button to match brand colors
- Include timestamp in log message prefix

## Bad Commit Messages

- Wrote some code
- it works now
- Changed style
- Bug fixed!
- commit message
