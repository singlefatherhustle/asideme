# Push

Conversely, we can **push** our commits to update the remote repository with the changes in our local repository.

Be careful! This will **overwrite** the remote repository. Always pull before you push!

## Diagram: Git Push Workflow

```
Local Repository (on laptop/computer)
            ↓
        git push
            ↓
GitHub (Remote) Repository (in cloud)
```

**Components shown:**
- **Local**: Repository folder icon with computer/laptop device
- **GitHub (Remote)**: Repository folder icon within a cloud shape
- **Arrow**: Labeled "git push" pointing from Local Repository to GitHub Remote Repository
- Both repositories are represented by yellow folder icons
